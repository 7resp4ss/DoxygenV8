Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1internal_1_1ConcurrentMarking_1_1JobTaskMajor_a49e85732d138fe1c698452897a121962}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1internal_1_1ConcurrentMarking_1_1JobTaskMajor_a49e85732d138fe1c698452897a121962}{Run()}}. \mbox{\hyperlink{classv8_1_1internal_1_1ConcurrentMarking_1_1JobTaskMajor_a49e85732d138fe1c698452897a121962}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1internal_1_1ConcurrentMarking_1_1JobTaskMajor_a49e85732d138fe1c698452897a121962}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1internal_1_1ConcurrentMarking_1_1JobTaskMajor_a33a3581b5cf68cb8c0bf32bb33068aeb}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

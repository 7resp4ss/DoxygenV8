Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1internal_1_1LazyCompileDispatcher_1_1JobTask_a852e4d34ab70d436d1194d6efd360676}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1internal_1_1LazyCompileDispatcher_1_1JobTask_a852e4d34ab70d436d1194d6efd360676}{Run()}}. 

\mbox{\hyperlink{classv8_1_1internal_1_1LazyCompileDispatcher_1_1JobTask_a852e4d34ab70d436d1194d6efd360676}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1internal_1_1LazyCompileDispatcher_1_1JobTask_a852e4d34ab70d436d1194d6efd360676}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1internal_1_1LazyCompileDispatcher_1_1JobTask_aa9e830a8e16267de19148a8064d4282e}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

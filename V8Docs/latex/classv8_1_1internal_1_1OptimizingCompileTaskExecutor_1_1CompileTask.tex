Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1internal_1_1OptimizingCompileTaskExecutor_1_1CompileTask_a020e41832e5933877dc122d234d57d70}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1internal_1_1OptimizingCompileTaskExecutor_1_1CompileTask_a020e41832e5933877dc122d234d57d70}{Run()}}. 

\mbox{\hyperlink{classv8_1_1internal_1_1OptimizingCompileTaskExecutor_1_1CompileTask_a020e41832e5933877dc122d234d57d70}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1internal_1_1OptimizingCompileTaskExecutor_1_1CompileTask_a020e41832e5933877dc122d234d57d70}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1internal_1_1OptimizingCompileTaskExecutor_1_1CompileTask_a1c6eda331c193bc75994b7212021465e}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

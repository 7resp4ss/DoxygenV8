Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1internal_1_1PageEvacuationJob_ab708e50f29b82e26827fe0da974196e7}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1internal_1_1PageEvacuationJob_ab708e50f29b82e26827fe0da974196e7}{Run()}}. \mbox{\hyperlink{classv8_1_1internal_1_1PageEvacuationJob_ab708e50f29b82e26827fe0da974196e7}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1internal_1_1PageEvacuationJob_ab708e50f29b82e26827fe0da974196e7}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1internal_1_1PageEvacuationJob_a5a339abfedcef84952b058b058769ecc}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

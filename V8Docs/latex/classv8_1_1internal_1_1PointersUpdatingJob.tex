Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1internal_1_1PointersUpdatingJob_a4bfd19244863f76434a26a28bd0224dd}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1internal_1_1PointersUpdatingJob_a4bfd19244863f76434a26a28bd0224dd}{Run()}}. 

\mbox{\hyperlink{classv8_1_1internal_1_1PointersUpdatingJob_a4bfd19244863f76434a26a28bd0224dd}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1internal_1_1PointersUpdatingJob_a4bfd19244863f76434a26a28bd0224dd}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1internal_1_1PointersUpdatingJob_a1e85c703f3a66ede1250d045df8717a3}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

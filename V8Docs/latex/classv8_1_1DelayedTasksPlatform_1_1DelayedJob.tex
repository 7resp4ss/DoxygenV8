Controls the maximum number of threads calling \mbox{\hyperlink{classv8_1_1DelayedTasksPlatform_1_1DelayedJob_a9fb75d700555f64567b57ff93dc63d09}{Run()}} concurrently, given the number of threads currently assigned to this job and executing \mbox{\hyperlink{classv8_1_1DelayedTasksPlatform_1_1DelayedJob_a9fb75d700555f64567b57ff93dc63d09}{Run()}}. 

\mbox{\hyperlink{classv8_1_1DelayedTasksPlatform_1_1DelayedJob_a9fb75d700555f64567b57ff93dc63d09}{Run()}} is only invoked if the number of threads previously running \mbox{\hyperlink{classv8_1_1DelayedTasksPlatform_1_1DelayedJob_a9fb75d700555f64567b57ff93dc63d09}{Run()}} was less than the value returned. In general, this should return the latest number of incomplete work items (smallest unit of work) left to process, including items that are currently in progress. $\vert$worker\+\_\+count$\vert$ is the number of threads currently assigned to this job which some callers may need to determine their return value. Since \mbox{\hyperlink{classv8_1_1DelayedTasksPlatform_1_1DelayedJob_aea4c91416067f8e60027bc6c0ba295f3}{Get\+Max\+Concurrency()}} is a leaf function, it must not call back any \mbox{\hyperlink{classv8_1_1JobHandle}{Job\+Handle}} methods.

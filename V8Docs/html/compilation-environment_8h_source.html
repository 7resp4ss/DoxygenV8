38 // The Arm architecture does not specify the results in memory of
39 // partially-in-bound writes, which does not align with the wasm spec. This
40 // affects when trap handlers can be used for OOB detection; however, Mac
41 // systems with Apple silicon currently do provide trapping beahviour for
42 // partially-out-of-bound writes, so we assume we can rely on that on MacOS,
43 // since doing so provides better performance for writes.

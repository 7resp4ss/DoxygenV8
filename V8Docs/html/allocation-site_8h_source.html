107  // A "zombie" AllocationSite is one which has no more strong roots to
108  // it, and yet must be maintained until the next GC. The reason is that
109  // it may be that in new space there are AllocationMementos hanging around
110  // which point to the AllocationSite. If we scavenge these AllocationSites
111  // too soon, those AllocationMementos will end up pointing to garbage
112  // addresses. The concrete case happens when evacuating new space in the full
113  // GC which happens after sweeping has been started already. To mitigate this
114  // problem the garbage collector marks such AllocationSites as zombies when it
115  // discovers there are no roots, allowing the subsequent collection pass to
116  // recognize zombies and discard them later.

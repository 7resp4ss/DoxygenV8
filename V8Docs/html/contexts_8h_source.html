433 // JSFunctions are pairs (context, function code), sometimes also called
434 // closures. A Context object is used to represent function contexts and
435 // dynamically pushed 'with' contexts (or 'scopes' in ECMA-262 speak).
436 //
437 // At runtime, the contexts build a stack in parallel to the execution
438 // stack, with the top-most context being the current context. All contexts
439 // have the following slots:
440 //
441 // [ scope_info ] This is the scope info describing the current context. It
442 // contains the names of statically allocated context slots,
443 // and stack-allocated locals. The names are needed for
444 // dynamic lookups in the presence of 'with' or 'eval', and
445 // for the debugger.
446 //
447 // [ previous ] A pointer to the previous context.
448 //
449 // [ extension ] Additional data. This slot is only available when
450 // ScopeInfo::HasContextExtensionSlot returns true.
451 //
452 // For native contexts, it contains the global object.
453 // For module contexts, it contains the module object.
454 // For await contexts, it contains the generator object.
455 // For var block contexts, it may contain an "extension
456 // object".
457 // For with contexts, it contains an "extension object".
458 //
459 // An "extension object" is used to dynamically extend a
460 // context with additional variables, namely in the
461 // implementation of the 'with' construct and the 'eval'
462 // construct. For instance, Context::Lookup also searches
463 // the extension object for properties. (Storing the
464 // extension object is the original purpose of this context
465 // slot, hence the name.)
466 //
467 // In addition, function contexts with sloppy eval may have statically
468 // allocated context slots to store local variables/functions that are accessed
469 // from inner functions (via static context addresses) or through 'eval'
470 // (dynamic context lookups).
471 // The native context contains additional slots for fast access to native
472 // properties.
473 //
474 // Finally, with Harmony scoping, the JSFunction representing a top level
475 // script will have the ScriptContext rather than a FunctionContext.
476 // Script contexts from all top-level scripts are gathered in
477 // ScriptContextTable.

77 // This interface allows to extend the GC to consider more roots during a GC
78 // cycle. This allows us to avoid the use of handles in C++ objects.
79 //
80 // During a GC the `Iterate()` method will be invoked with the GC's RootVisitor
81 // passed in as argument. The implementer of this interface needs to invoke the
82 // visitor on each unhandlified pointer in the object. The GC will both keep the
83 // referenced object alive and also updates its address if the object gets
84 // relocated. Note that in order to do so `Iterate()` may be invoked more than
85 // once per GC.
86 //
87 // Note that an object implementing GCRootsProvider still needs to be registered
88 // with a thread (=LocalHeap/LocalIsolate) using GCRootsProviderScope.
89 class GCRootsProvider {
90  public:
91  virtual ~GCRootsProvider() {}
92 
93  virtual void Iterate(RootVisitor* v) = 0;
94 };

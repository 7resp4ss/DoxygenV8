198 // The number of calls to an exported Wasm function that will be handled
199 // by the generic wrapper. Once the budget is exhausted, a specific wrapper
200 // is to be compiled for the function's signature.
201 // The abstract goal of the tiering strategy for the js-to-wasm wrappers is to
202 // use the generic wrapper as much as possible (less space, no need to compile),
203 // but fall back to compiling a specific wrapper for any function (signature)
204 // that is used often enough for the generic wrapper's small execution penalty
205 // to start adding up.
206 // So, when choosing a value for the initial budget, we are interested in a
207 // value that skips on tiering up functions that are called only a few times and
208 // the tier-up only wastes resources, but triggers compilation of specific
209 // wrappers early on for those functions that have the potential to be called
210 // often enough.
211 constexpr uint32_t kGenericWrapperBudget = 1000;

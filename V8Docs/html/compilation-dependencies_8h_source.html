55  // Depend on the fact that accessing property |property_name| from
56  // |receiver_map| yields the constant value |constant|, which is held by
57  // |holder|. Therefore, must be invalidated if |property_name| is added to any
58  // of the objects between receiver and |holder| on the prototype chain, b) any
59  // of the objects on the prototype chain up to |holder| change prototypes, or
60  // c) the value of |property_name| in |holder| changes.
61  // If PropertyKind is kData, |constant| is the value of the property in
62  // question. In case of PropertyKind::kAccessor, |constant| is the accessor
63  // function (i.e., getter or setter) itself, not the overall AccessorPair.
64  void DependOnConstantInDictionaryPrototypeChain(MapRef receiver_map,
65  NameRef property_name,
66  ObjectRef constant,
67  PropertyKind kind);

73  // Return a field's constness and, if kConst, record the assumption that it
74  // remains kConst. The field is identified by the arguments.
75  //
76  // For arrays, arguments objects and value wrappers, only consider the field
77  // kConst if the map is stable (and register stability dependency in that
78  // case). This is to ensure that fast elements kind transitions cannot be
79  // used to mutate fields without deoptimization of the dependent code.

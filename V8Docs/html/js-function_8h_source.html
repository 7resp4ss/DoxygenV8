225  // Tiering up a function happens as follows:
226  // 1. RequestOptimization is called
227  // -> From now on `IsOptimizationRequested` and also
228  // `IsTieringRequestedOrInProgress` return true.
229  // 2. On the next function invocation the optimization is triggered. While the
230  // optimization progresses in the background both
231  // `IsTieringRequestedOrInProgress` and `tiering_in_progress` return
232  // true. It also means the optimization is no longer requested (i.e.,
233  // `IsOptimizationRequested` returns false).
234  // 3. Once the compilation job is finalized the functions code is installed
235  // via `UpdateCode` and any remaining flags cleared by
236  // `ResetTieringRequests`.
237  // NB: Osr tiering state is tracked separately from these.
238

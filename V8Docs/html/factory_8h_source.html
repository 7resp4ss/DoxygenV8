260  // String creation functions. Most of the string creation functions take
261  // an AllocationType argument to optionally request that they be
262  // allocated in the old generation. Otherwise the default is
263  // AllocationType::kYoung.
264  //
265  // Creates a new String object. There are two String encodings: one-byte and
266  // two-byte. One should choose between the three string factory functions
267  // based on the encoding of the string buffer that the string is
268  // initialized from.
269  // - ...FromOneByte (defined in FactoryBase) initializes the string from a
270  // buffer that is Latin1 encoded (it does not check that the buffer is
271  // Latin1 encoded) and the result will be Latin1 encoded.
272  // - ...FromUtf8 initializes the string from a buffer that is UTF-8
273  // encoded. If the characters are all ASCII characters, the result
274  // will be Latin1 encoded, otherwise it will converted to two-byte.
275  // - ...FromTwoByte initializes the string from a buffer that is two-byte
276  // encoded. If the characters are all Latin1 characters, the result
277  // will be converted to Latin1, otherwise it will be left as two-byte.
278  //
279  // One-byte strings are pretenured when used as keys in the SourceCodeCache.
280  template <size_t N>
281  inline Handle<String> NewStringFromStaticChars(
282  const char (&str)[N], AllocationType allocation = AllocationType::kYoung);

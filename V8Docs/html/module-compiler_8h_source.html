105 // Encapsulates all the state and steps of an asynchronous compilation.
106 // An asynchronous compile job consists of a number of tasks that are executed
107 // as foreground and background tasks. Any phase that touches the V8 heap or
108 // allocates on the V8 heap (e.g. creating the module object) must be a
109 // foreground task. All other tasks (e.g. decoding and validating, the majority
110 // of the work of compilation) can be background tasks.
111 // TODO(wasm): factor out common parts of this with the synchronous pipeline.

440 // Auxiliary class for translating deoptimization values.
441 // Typical usage sequence:
442 //
443 // 1. Construct the instance. This will involve reading out the translations
444 // and resolving them to values using the supplied frame pointer and
445 // machine state (registers). This phase is guaranteed not to allocate
446 // and not to use any HandleScope. Any object pointers will be stored raw.
447 //
448 // 2. Handlify pointers. This will convert all the raw pointers to handles.
449 //
450 // 3. Reading out the frame values.
451 //
452 // Note: After the instance is constructed, it is possible to iterate over
453 // the values eagerly.

274 // V8_HEAP_USE_PTHREAD_JIT_WRITE_PROTECT controls how V8 sets permissions for
275 // executable pages.
276 // In particular,
277 // 1) when memory region is reserved for code range, the whole region is
278 // committed with RWX permissions and then the whole region is discarded,
279 // 2) since reconfiguration of RWX page permissions is not allowed on MacOS on
280 // ARM64 ("Apple M1"/Apple Silicon), there must be no attempts to change
281 // them,
282 // 3) the request to set RWX permissions in the execeutable page region just
283 // commits the pages without changing permissions (see (1), they were already
284 // allocated as RWX and then deommitted),
285 // 4) in order to make executable pages inaccessible one must use
286 // OS::DiscardSystemPages() instead of using OS::DecommitPages() or setting
287 // permissions to kNoAccess because the latter two are not allowed by the
288 // MacOS (see (2)).
289 //
290 // This is applicable only to MacOS on ARM64 ("Apple M1"/Apple Silicon) which
291 // has a APRR/MAP_JIT machinery for fast W^X permission switching (see
292 // pthread_jit_write_protect).
293 //
294 // This approach doesn't work and shouldn't be used for V8 configuration with
295 // enabled pointer compression and disabled external code space because
296 // a) the pointer compression cage has to be reserved with MAP_JIT flag which
297 // is too expensive,
298 // b) in case of shared pointer compression cage if the code range will be
299 // deleted while the cage is still alive then attempt to configure
300 // permissions of pages that were previously set to RWX will fail.
301 //
302 #if V8_HAS_PTHREAD_JIT_WRITE_PROTECT && \
303  !(defined(V8_COMPRESS_POINTERS) && !defined(V8_EXTERNAL_CODE_SPACE))
304 #define V8_HEAP_USE_PTHREAD_JIT_WRITE_PROTECT true
305 #else
306 #define V8_HEAP_USE_PTHREAD_JIT_WRITE_PROTECT false
307 #endif
